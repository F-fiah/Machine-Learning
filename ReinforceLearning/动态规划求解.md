# 求解方案
## Policy Iteration

政策评估 —> 政策改进
$$v_{k+1}(s) = \sum_{a \in A} \pi(a|s) \left( R_s^a + \gamma \sum_{s' \in S} P_{ss'}^a v_k(s') \right)$$
计算当前政策 $\pi$ 下的价值函数 $v_\pi(s)$ 
![[政策迭代.png]]
## Value Iteration

对价值函数进行迭代，中间迭代得到的 $v_k$ 不一定对应于一个真实的政策。当价值函数收敛后，才通过贪心决策提取到 $\pi^*$ 
$$v_{k+1}(s) = \max_{a \in A} \left( R_s^a + \gamma \sum_{s' \in S} P_{ss'}^a v_k(s') \right)$$
取所有动作的最大值

![[价值迭代.png]]

### 同步迭代（Synchronous Value Iteration）
$$v_new(s) ← max_{a∈A} ( R_s^a + γ ∑_{s'∈S} P_{ss'}^a v_old(s') )$$
$$v_old ← v_new$$
存储两份价值函数，依次更新

### 原地迭代（In-place Value Iteration）
$$v(s) ← max_{a∈A} ( R_s^a + γ ∑_{s'∈S} P_{ss'}^a v(s') )$$
按某种顺序逐个更新状态的价值

**如何排序使算法更efficient** : Discuss later
 

**两种算法的过程不同，但结果相同，都会收敛到optimal policy**
$\pi_0$ 和 $v_0$ 是可以随意设置的（通常情况下全部设置为0），**迭代的收敛性不依赖初始值**
