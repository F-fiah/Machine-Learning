# 模型评估

- 过拟合：泛化性能下降，高方差
- 欠拟合：高偏差，未提取出样本的大部分 feature
当模型过于复杂时，泛化能力往往会下降
# 正则化

防止模型过拟合的一种方法

以（广义）线性回归为例
目标为：$\min_{\theta}\ \dfrac12\sum_{i=1}^m \left\|y^{(i)}-\theta^\mathrm{T}x^{(i)}\right\|^2$
将其修正为：$\min_{\theta}\ \dfrac12\sum_{i=1}^m \left\|y^{(i)}-\theta^\mathrm{T}x^{(i)}\right\|^2+\dfrac{\lambda}{2}\|\theta\|^2$
过拟合的情况下某些参数往往会很大，所以加上这个修正项后可以使模型更平滑

**注意：**
1. 正则化这个方法是在模型的方程（如维数）确定的情况下使用的，无法筛选不同的模型
2. 支持向量机不会过拟合
3. 正则化本质上也是惩罚型优化框架，和[[支持向量机(Support Vector Machine)#Soft Margin]]很类似
4. 当样本数量相比于参数数量比较大时（10倍），不使用正则化才会有比较好的效果，因为这样才有足够的信息去选择合适的参数。否则，应当使用正则化
# 模型评估
## 交叉验证

将数据 S 分成三部分：$S_{train},S_{dev},S_{test}$
对 $S_{train}$ 使用不同的模型，计算出模型的参数
用 $S_{dev}$ 测量误差，选择误差最小的
用 $S_{test}$ 进一步评估模型的泛化能力 

当数据总量较小时：
$60\%S_{train} + 20\%S_{dev} + 20\%S_{test}$
当数据总量较大时：
$S_{dev}$ 和 $S_{test}$ 的量应当适当增加，总而可以更好地比较出不同模型之间的微小差异
# k-fold 交叉验证

适用于数据量非常小的情况
将数据分成 k 份（k 通常为10）
```
for each model
	for i=1...k
		do test on the k piece
		train on the remaining data
	end
end
take average of the error(varriance)
choose the best model
```
# 特征选择
## 前向搜索

减少过拟合的一种方法使找到合适的 feature
```
Start with ℱ = ∅ 
Repeat
	Try adding each feature i to ℱ
	see which single-feature addition most improves dev set performance
	Add that feature to ℱ
	(Optional) Stop when no feature addition improves dev set performance.
end
```
